## ◈ Mathematical Foundation

> Built on **rigorous thermodynamic and information-theoretic principles** — stripped of speculative hype (Phase 10 Reality Check).

---

### 1 · State Superposition

Solutions exist as a **probability field** over the full candidate space — not a single point.

```
|Ψ_t⟩  =  Σᵢ αᵢ |sᵢ⟩
```

```
  αᵢ   →  probability amplitude of solution sᵢ
  |sᵢ⟩ →  basis state (candidate solution)
```

---

### 2 · Probabilistic State Evolution (Boltzmann)

States evolve via a Boltzmann-like distribution, **dynamically reshaped by entropy**:

```
         Pᵢ(t) · exp( −β(t) · Eᵢ(t) )
Pᵢ(t+1) = ─────────────────────────────────
            Σⱼ Pⱼ(t) · exp( −β(t) · Eⱼ(t) )
```

```
  Pᵢ(t)   →  probability of solution i at time t
  β(t)    →  inverse temperature (amplification)
  Eᵢ(t)   →  energy of solution i
```

---

### 3 · Shannon Entropy Monitor

Tracks **population diversity** in real time — the system's search breadth signal:

```
H(t)  =  −Σᵢ Pᵢ(t) · log Pᵢ(t)
```

```
  H(t) → HIGH  =  broad exploration (diverse population)
  H(t) → LOW   =  focused exploitation (converging)
```

**Entropy Damping** — smooths oscillations:

```
H'(t)  =  (1 − μ) · H(t)  +  μ · H(t−1)
```

---

### 4 · Entropy-Regulated Amplification

β self-adjusts to keep search diversity near target — **no manual schedule needed**:

```
β(t+1)  =  β(t)  +  η · ( H_target − H(t) )
```

```
  η         →  learning rate for amplification
  H_target  →  desired entropy level
```

---

### 5 · Consolidated Energy Function (Phase 10)

Reduced to **three essential components** for stability and tractability:

```
Eᵢ  =  wq · Qᵢ  +  wd · Dᵢ  +  wc · Cᵢ
```

```
  Qᵢ   →  solution quality       (objective cost)
  Dᵢ   →  diversity penalty      (similarity to existing solutions)
  Cᵢ   →  computational cost     (evaluation expense)
  wq, wd, wc  →  learnable weights
```

**Full AQSE-v2 Energy** (Phases I–IX):

```
Eᵢ  =  w₁·Cᵢ + w₂·Dᵢ + w₃·Rᵢ + w₄·Vᵢ + w₅·Nᵢ
```

```
  Cᵢ   →  objective cost          Vᵢ  →  quality volatility
  Dᵢ   →  similarity density      Nᵢ  →  stochastic noise
  Rᵢ   →  instability risk
```

---

### 6 · Dynamic Collapse Gate θ(t)

Controls **when** a solution collapses from superposition to a definite state:

```
θ(t)  =  θ₀  +  α · ( 1  −  H(t) / H_max )
```

```
  θ(t) RISES as entropy FALLS  →  stricter collapse criterion near convergence
  θ(t) FALLS as entropy RISES  →  permissive collapse during exploration
```

---

### 7 · Entropy-Triggered Regeneration

Injects diversity when search stagnates — the system's **self-healing mechanism**:

```
         ρ · σ_H(t)
G(t)  =  ──────────────
          σ_H(t)  +  ε
```

```
  σ_H(t)  →  standard deviation of recent entropy
  ρ       →  regeneration strength
  ε       →  stability constant (prevents division by zero)
```

---

### 8 · AI Reward Signal + Learning Potential

Guides evolution with **intelligence-driven scoring** beyond raw objective cost:

```
Rᵢ  =  γ₁·Qᵢ  −  γ₂·Cᵢ  −  γ₃·Vᵢ  +  γ₄·Lᵢ
```

**Learning Potential** — rewards solutions improving fastest:

```
        ΔQᵢ
Lᵢ  =  ──────
        Δt + ε
```

---

### 9 · Turbulence Monitor T(t)

Detects **phase transitions** in search dynamics:

```
T(t)  =  | H(t)  −  H(t−1) |
```

```
  T(t) → HIGH  =  search is destabilizing (intervention needed)
  T(t) → LOW   =  search is stable (continue current regime)
```

---

### 10 · Lyapunov Stability Guarantee

Convergence is **mathematically proven** — not just empirically observed:

```
V(t)  =  E(t)  +  λ · H(t)
```

```
Stability condition:   dV/dt  <  0   for all t
```

```
  V(t) is a valid Lyapunov function if it decreases monotonically
  →  guaranteed convergence to a fixed point
  λ   →  entropy weight in the composite Lyapunov potential
```

## ◈ Quick Start

```bash
git clone https://github.com/GypsianMonk/HAQOA.git
cd HAQOA
pip install -r requirements.txt

# GPU support (CUDA 11.8)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
```

```bash
# Run HAQOA-X on 20-city instance
python run_haqoax.py --instance small --iters 300 --pop 60

# Run on 50-city clustered instance
python run_haqoax.py --instance medium --iters 500 --pop 80

# Sweep all benchmark instances
python run_haqoax.py --compare --iters 300

# Phase 3 statistical validation (15 runs)
python run_phase3.py --instance small --runs 15 --iters 300
```
